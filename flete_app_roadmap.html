Estimado
Fase 1 Entorno y estructura base ~1-2 días ▼
1
Instalar herramientas en tu PC
Python 3.11+, Flutter SDK, Node.js, PostgreSQL local, VS Code, Git, Docker (opcional)
Tu computador — instalador oficial de cada tool
2
Crear repositorio Git
Un repo con dos carpetas raíz: /backend y /mobile. Crea en GitHub o GitLab.
GitHub.com → New repository
3
Crear base de datos PostgreSQL local
Instala PostgreSQL, crea la BD fleteapp_db y usuario fleteapp_user.
Terminal local → psql
4
Crear proyecto FastAPI
Dentro de /backend: crea virtualenv, instala FastAPI, SQLAlchemy, Alembic, Pydantic, python-jose, passlib, httpx.
/backend → terminal
5
Crear proyecto Flutter
Dentro de /mobile: flutter create flete_app. Agrega dependencias: dio, provider o riverpod, google_maps_flutter, geolocator, firebase_messaging.
/mobile → terminal
Fase 2 Backend — Modelos, Auth y CRUD base ~3-5 días ▼
6
Crear estructura de carpetas del backend
app/models, app/schemas, app/routers, app/services, app/core. Crear main.py, database.py, .env.
/backend/app → VS Code
7
Definir modelos SQLAlchemy
Tablas: users, drivers, vehicles, freight_requests, trip_status_history, payments, ratings, notifications.
/backend/app/models/
8
Configurar Alembic y migrar BD
Inicializar Alembic, crear migración inicial, correr alembic upgrade head.
/backend → terminal
9
Implementar autenticación JWT
Hash de contraseñas con bcrypt, generación y validación de tokens, dependencia get_current_user, roles (cliente, conductor, admin).
/backend/app/core/security.py
10
Crear endpoints de auth y usuarios
POST /auth/register, POST /auth/login, GET /users/me, PUT /users/me, GET /users/{id} (admin).
/backend/app/routers/auth.py, users.py
11
Crear endpoints de conductores
POST /drivers/register, GET /drivers/me, PUT /drivers/me, POST /drivers/vehicle, GET /drivers/available.
/backend/app/routers/drivers.py
12
Crear endpoints de fletes
POST /freights, GET /freights, GET /freights/{id}, PUT /freights/{id}/accept, PUT /freights/{id}/status, GET /freights/{id}/history.
/backend/app/routers/freights.py
13
Lógica de negocio: tarifas y estados
Servicio de estimación de precio (distancia × tarifa base + peso). Máquina de estados: PENDING → ACCEPTED → IN_PROGRESS → COMPLETED / CANCELLED.
/backend/app/services/freight_service.py
14
Endpoints de pagos y calificaciones
POST /payments/initiate, POST /payments/callback, GET /payments/{id}, POST /ratings, GET /ratings/driver/{id}.
/backend/app/routers/payments.py, ratings.py
15
Probar con Postman o Swagger
FastAPI genera docs en /docs automáticamente. Prueba todos los endpoints antes de pasar al frontend.
Navegador → http://localhost:8000/docs
Fase 3 App Flutter — Pantallas y navegación ~5-8 días ▼
16
Estructura de carpetas Flutter
lib/screens, lib/widgets, lib/services, lib/models, lib/providers. Configurar Riverpod o Provider.
/mobile/flete_app/lib/
17
Servicio HTTP y manejo de tokens
Clase ApiService con Dio, interceptor para adjuntar JWT, manejo de errores 401/403, guardar token en secure storage.
/lib/services/api_service.dart
18
Pantallas de auth: Login y Registro
Formularios validados, selector de rol (cliente / conductor), llamada a API, redirección según rol.
/lib/screens/auth/
19
Pantallas del cliente
HomeCliente, CrearFlete (con mapa), ListaFletes, DetalleFlete, Historial, Perfil, Calificación.
/lib/screens/client/
20
Pantallas del conductor
HomeConductor, FleteDisponible, AceptarFlete, ActualizarEstado, HistorialViajes, Perfil, DatosVehiculo.
/lib/screens/driver/
21
Integrar Google Maps
Obtener API key en Google Cloud Console, configurar en AndroidManifest y Info.plist, widget de mapa, selección de origen/destino, cálculo de ruta con Directions API.
console.cloud.google.com + AndroidManifest.xml
22
Configurar Firebase Cloud Messaging
Crear proyecto en Firebase Console, descargar google-services.json (Android) y GoogleService-Info.plist (iOS), integrar firebase_messaging.
console.firebase.google.com + /android/app/
Fase 4 Pagos, Admin y Seguridad ~3-4 días ▼
23
Integrar Webpay (Transbank) en sandbox
Instalar SDK transbank-sdk, configurar credenciales sandbox, endpoint de inicio de pago y callback, manejar respuesta de Transbank.
/backend/app/services/payment_service.py
24
Panel admin — endpoints o web simple
GET /admin/users, GET /admin/drivers, PUT /admin/users/{id}/suspend, GET /admin/trips, GET /admin/payments, GET /admin/metrics. Protegidos por rol admin.
/backend/app/routers/admin.py
25
Validación de inputs y seguridad
Validadores Pydantic en todos los schemas, rate limiting con slowapi, CORS configurado, variables de entorno con python-decouple.
/backend/app/core/
Fase 5 Despliegue y producción ~2-3 días ▼
26
Desplegar backend en Railway o Render
Crear cuenta en Railway.app o Render.com, conectar repo GitHub, agregar PostgreSQL como servicio, configurar variables de entorno, deploy automático.
railway.app o render.com
27
Subir imágenes/documentos a Cloudinary o S3
Cuenta gratuita en Cloudinary para fotos de perfil, vehículos y documentos. Integrar en el backend con cloudinary Python SDK.
cloudinary.com + /backend/app/services/storage_service.py
28
Build y distribución de app Flutter
Actualizar URL base de API al backend en producción. flutter build apk para Android. Para iOS necesitas Mac + cuenta Apple Developer ($99/año). Distribuir con Firebase App Distribution para pruebas.
/mobile → terminal + Firebase Console
Haz clic en cada fase para expandir o colapsar. Toca cualquier paso para pedir más detalles.
